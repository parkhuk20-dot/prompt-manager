# 프롬프트 관리 프로그램 - 과제 제출 문서

## 1. 프로젝트 개요

GenAI 프롬프트(텍스트/이미지/영상/페르소나/자동화 등)를 카테고리, 검색, 즐겨찾기, 조회수 기준으로 관리하는 **콘솔 기반 Python 프로그램**입니다. 파이썬 기초 문법(변수, 조건문, 반복문, 함수, 리스트/딕셔너리)과 Git/GitHub를 활용한 버전 관리를 실습하기 위해 제작했습니다.

- **GitHub 저장소**: https://github.com/parkhuke20-dot/prompt-manager
- **개발 환경**: macOS, VSCode, Python 3.12.12, Git 2.50.1
- **파일 구성**: `prompt_manager.py`(메인 프로그램), `README.md`, `.gitignore`, `SUBMISSION.md`

## 3. 실행 방법

Python 3.10 이상이 설치되어 있어야 합니다.

```bash
python3 prompt_manager.py
```

실행 후 메뉴에 표시되는 번호를 입력해 원하는 기능을 선택합니다.

### 입력 단축키

| 단축키 | 동작 |
|---|---|
| 숫자 | 해당 번호 선택 |
| `b` | 이전 화면으로 이동 |
| `0` | 전체 메뉴로 이동 |
| `/s` | 여러 줄 입력 저장 (추가/수정 시) |
| `a` | 전체 선택 (JSON/Markdown 내보내기 시) |

## 4. 기능 목록

### 기본 기능 (필수 요구사항)

| 번호 | 기능 | 설명 |
|---|---|---|
| 1 | 프롬프트 추가 | 제목, 내용(여러 줄 입력 가능), 카테고리를 입력해 새 프롬프트 등록 |
| 2 | 프롬프트 목록 | 전체 프롬프트를 번호, 카테고리, 즐겨찾기 여부와 함께 출력 |
| 3 | 카테고리별 조회 | 선택한 카테고리에 속한 프롬프트만 조회 |
| 4 | 프롬프트 검색 | 제목 또는 내용에 포함된 키워드로 검색 |
| 5 | 프롬프트 상세 보기 | 번호 선택 시 전체 내용 및 조회수 출력 |
| 6 | 즐겨찾기 관리 | 프롬프트 번호 선택으로 즐겨찾기 추가/해제 (토글) |
| 7 | 즐겨찾기 목록 | 즐겨찾기한 프롬프트만 모아서 조회 |

### 보너스 기능

| 번호 | 기능 | 설명 |
|---|---|---|
| 8 | JSON으로 저장 | 선택한 프롬프트를 JSON 파일로 저장 (단일/복수/전체 선택, 누적 저장 지원) |
| 9 | JSON 불러오기 | 저장된 JSON 파일에서 프롬프트 불러오기 |
| 10 | Markdown으로 내보내기 | 선택한 프롬프트를 카테고리별 Markdown 파일로 내보내기 |
| 11 | 프롬프트 수정 | 제목, 내용, 카테고리 수정 (Enter로 유지, 현재 내용 확인 후 수정 가능) |
| 12 | 프롬프트 삭제 | 확인 후 프롬프트 삭제 |
| 13 | 조회수 Top 목록 | 상세 보기 횟수 기준으로 정렬된 목록 조회 |

### 검색 동작 규칙

- 프롬프트의 **제목과 내용 모두**를 대상으로 검색합니다.
- **부분 매칭**을 지원합니다. 예를 들어 "블로그"로 검색하면 "블로그 글 작성 도우미"가 검색됩니다.
- 파이썬 문자열의 `in` 연산자를 사용하므로 입력한 키워드가 그대로 포함된 항목을 찾습니다(대소문자 구분).

## 7. 자료구조 설계 (리스트 + 딕셔너리)

이 프로그램은 프롬프트 데이터를 **리스트 안에 딕셔너리를 담는 구조**(`list[dict]`)로 설계했습니다.

```python
prompts = [
    {
        "title": "블로그 글 작성 도우미",
        "content": "당신은 10년 경력의 전문 블로거입니다...",
        "category": "텍스트 생성",
        "favorite": False,
        "views": 0,
    },
    ...
]
```

### 왜 리스트(list)를 사용했는가

- 프롬프트는 **순서가 있는 여러 개의 항목**이므로, 순서를 유지하고 번호(인덱스)로 접근할 수 있는 리스트가 적합합니다.
- 목록을 번호와 함께 출력하거나, 사용자가 번호로 특정 프롬프트를 선택할 때 인덱스 접근이 직관적입니다.
- 추가(`append`), 삭제(`pop`), 반복(`for`), 필터링(리스트 컴프리헨션)이 간단합니다.

### 왜 딕셔너리(dict)를 사용했는가

- 하나의 프롬프트는 제목, 내용, 카테고리, 즐겨찾기, 조회수 등 **여러 속성을 가진 하나의 객체**입니다.
- 딕셔너리는 `p["title"]`처럼 **의미 있는 키(key)로 값에 접근**할 수 있어 가독성이 높고 유지보수가 쉽습니다.
- 속성을 추가/변경하기 쉽습니다(예: 조회수 `views` 필드를 나중에 추가).

### 장단점 요약

| 구조 | 장점 | 단점 |
|---|---|---|
| 리스트 | 순서 유지, 번호 접근, 반복/필터 용이 | 특정 항목 검색 시 전체 순회 필요(O(n)) |
| 딕셔너리 | 키 기반 명확한 접근, 속성 확장 용이 | 프롬프트 간 순서 개념은 없음(리스트로 보완) |

두 구조를 결합함으로써 "순서 있는 여러 개의 프롬프트(리스트)"와 "속성이 많은 개별 프롬프트(딕셔너리)"를 모두 자연스럽게 표현할 수 있습니다.

## 8. JSON 저장 파일 형식 (prompts.json)

프롬프트 데이터는 `prompts.json` 파일에 UTF-8 인코딩으로 저장됩니다.

### 필드(키) 설명

| 키 | 타입 | 설명 |
|---|---|---|
| `title` | 문자열 | 프롬프트 제목 |
| `content` | 문자열 | 프롬프트 본문 내용 |
| `category` | 문자열 | 카테고리명 |
| `favorite` | 불리언 | 즐겨찾기 여부 (true/false) |
| `views` | 정수 | 상세 보기 조회수 |

### 중복 처리 정책

- JSON 저장 시 **이미 저장된 파일이 있으면 불러와서 누적 저장**합니다(덮어쓰기 아님).
- 제목(`title`)이 이미 존재하는 프롬프트는 **중복 저장하지 않고 스킵**하며, 스킵된 항목을 사용자에게 안내합니다.
- 백업/복원: `prompts.json` 파일을 복사해두면 백업이 되고, 메뉴 9번(JSON 불러오기)으로 복원할 수 있습니다.

## 10. Git / GitHub 활용 내역

### 사용한 Git 명령어

| 명령어 | 사용 목적 |
|---|---|
| `git init` | 로컬 저장소 초기화 |
| `git add` | 변경사항 스테이징 |
| `git commit` | 변경사항 커밋(기능 단위) |
| `git push` | GitHub 원격 저장소 업로드 |
| `git pull` | 원격 저장소 최신 내용 받기(노트북/부트캠프 PC 동기화) |
| `git clone` | 다른 컴퓨터에서 저장소 복제 |
| `git checkout` | 브랜치 생성 및 전환 |
| `git merge` | 브랜치 병합 |

### git clone 실행 내역

부트캠프 컴퓨터에서 작업을 이어가기 위해 저장소를 복제했습니다.

```bash
git clone https://github.com/parkhuke20-dot/prompt-manager.git
cd prompt-manager
```

복제 후 `git log --oneline`으로 노트북에서 작업한 전체 커밋 히스토리가 그대로 내려온 것을 확인했습니다. 실제 실행 로그 스크린샷은 11-9 섹션에 첨부합니다.

### 브랜치 생성 및 병합 기록

`feature/show-list` 브랜치를 생성하여 "프롬프트 목록" 기능을 개발한 뒤, `main` 브랜치로 병합했습니다.

```bash
git checkout -b feature/show-list   # 브랜치 생성 및 전환
# (목록 기능 개발 및 커밋)
git checkout main                    # main으로 이동
git merge feature/show-list          # 병합 (Fast-forward)
```

> 병합은 Fast-forward 방식으로 처리되어 별도의 충돌(conflict)은 발생하지 않았습니다.
